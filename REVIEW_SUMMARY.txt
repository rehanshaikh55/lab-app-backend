CRITICAL ISSUES (MUST FIX)
================================================================================

[CRITICAL-1] FirebaseStorageAdapter Not Exported
File: integrations/storage/storage.js:37
Issue: Class instantiated but not exported; controllers try to import it
Impact: Runtime failure on report uploads
Fix: Add "export" to class declaration

[CRITICAL-2] RBAC Middleware Missing Return Statement
File: middlewares/rbacMiddleware.js:11
Issue: Fastify preHandler must return; missing return causes requests to hang
Impact: All /partner and role-protected routes silently fail
Fix: Add "return;" after permission check success

[CRITICAL-3] Mongoose Subdocument Query Bug
File: controllers/bookingController.js:62
Issue: Uses non-existent .addresses.id() method
Impact: HOME collection bookings crash when fetching address
Fix: Replace with .addresses.find(a => a._id.toString() === id)

================================================================================
HIGH-PRIORITY ISSUES (SHOULD FIX)
================================================================================

[HIGH-1] Missing Try/Catch in 20+ Routes
Files: profileController, subscriptionController, labController, partnerController
Impact: Unhandled database errors crash requests
Affected: 20+ async route handlers
Fix: Wrap all DB operations in try/catch blocks

[HIGH-2] Configuration Duplication
Files: config/env.js vs config/config.js
Issue: PORT and COOKIE_PASSWORD exported twice with different logic
Impact: Confusion about which is authoritative; hardcoded cookie password
Fix: Single source of truth—remove config.js exports

[HIGH-3] JWT_SECRET Not Validated on Startup
File: config/env.js:3
Issue: JWT_SECRET has no fallback; could be undefined
Impact: Token verification silently fails
Fix: Add validation that JWT_SECRET is set

================================================================================
MEDIUM-PRIORITY ISSUES (NICE TO FIX)
================================================================================

[MEDIUM-1] console.log Used Instead of request.log
Files: config (3×), firebase.js (3×), storage.js (1×)
Impact: Logging inconsistency, hard to search logs
Fix: Replace console.log with request.log or app.log

[MEDIUM-2] Missing Input Validation Schemas
Files: partnerRoutes (3 routes), profileRoutes (1 route)

Fix: Add JSON schema validation blocks

[MEDIUM-3] Firebase Initialization Race Condition
File: config/firebase.js:6
Issue: Concurrent calls could initialize twice
Impact: Resource waste, potential SDK issues
Fix: Use promise-based initialization pattern

[MEDIUM-4] Subscription Job Doesn't Validate Lab Exists
File: jobs/subscriptions.js:65
Issue: Creates booking for deleted lab
Impact: Orphaned bookings with null lab reference
Fix: Cancel subscription if lab is deleted
